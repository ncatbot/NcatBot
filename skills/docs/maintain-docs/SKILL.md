# maintain-docs — 维护 NcatBot 文档

## 触发条件

当用户请求新增文档、修改文档、重构文档结构、同步 API 变更到文档，或要求检查文档完整性时，使用此技能。

---

## 使用前必读

> 完整规范见 [docs/meta/README.md](docs/meta/README.md)

关键约束速查：

- 单文件 ≤ 400 locs（含空行）；目标 < 200 locs
- 每新增/删除文件，必须同步更新父目录 `README.md` 和 `docs/README.md`
- 相对路径链接，禁止硬编码站点绝对路径
- 代码块必须标注语言（`python` / `toml` / `bash`）

---

## Step 1：定位正确的目录

根据文档类型选择目录：

| 文档类型 | 目录 | 判断标准 |
|----------|------|----------|
| "如何做 X" 教程 | `docs/guide/` | 任务导向，有步骤，有示例 |
| 类/函数 API 详解 | `docs/reference/` | 参考导向，有签名、参数表格 |
| 内部实现/设计决策 | `docs/contributing/` | 面向框架贡献者 |
| 系统全局架构 | `docs/architecture.md` | 跨模块视图 |

在 `guide/` 和 `reference/` 中按主题选择子目录（`plugin/`、`api/`、`events/` 等），参见 `docs/README.md` 目录树。

---

## Step 2：确定文件名

1. 查看目标目录下已有文件的命名风格
2. 有顺序的系列文档 → `数字前缀.主题.md`（如 `3a.loading.md`）
3. 独立参考文档 → `数字_主题.md`（如 `1a_message_api.md`）
4. 如新主题需要独立子目录，在目录名下创建 `README.md`

---

## Step 3：选择文档模板并起草内容

### 模板 A：指南文档（guide/）

```markdown
# 标题（动词短语）

> 一句话：本文档教你做什么，适用场景是什么。

## 前提条件

- 已完成 [上一步](../前一篇.md)
- 已安装 ...

## 基础用法

... 最常用的 20% 功能 ...

```python
# 可直接复制运行的示例
from ncatbot.plugin import NcatBotPlugin
```

## 进阶用法（可选）

... 较复杂的场景 ...

## 延伸阅读

- [相关参考文档](../../reference/.../xxx.md)
- [下一篇指南](下一篇.md)
```

### 模板 B：参考文档（reference/）

```markdown
# 模块/类名 参考

> 一句话：本文档描述什么。

## 类/函数签名

```python
class Foo:
    def bar(self, x: int, *, opt: str = "default") -> None: ...
```

## 参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `x` | `int` | — | 必填，描述 |
| `opt` | `str` | `"default"` | 可选，描述 |

## 示例

```python
foo = Foo()
foo.bar(42)
```

## 注意事项

- 边界情况 / 异常行为
```

### 模板 C：子目录 README.md

```markdown
# 子目录名 — 一句话说明

> 面向哪类读者，覆盖什么内容

## 快速开始

最短路径：3 步完成最常见操作...

## 文档清单

| 文件 | 说明 | 难度 |
|------|------|------|
| [1.xxx.md](1.xxx.md) | 描述 | ⭐ |
| [2.xxx.md](2.xxx.md) | 描述 | ⭐⭐ |

## 推荐阅读顺序

1. 先读 [1.xxx.md](1.xxx.md)
2. 再读 [2.xxx.md](2.xxx.md)
```

---

## Step 4：同步更新索引

新增文件后，**必须**更新以下位置：

### 4.1 父目录 README.md

在"文档清单"表格中新增一行：

```markdown
| [新文件名.md](新文件名.md) | 一句话说明 | 难度/篇数 |
```

并在"推荐阅读顺序"或"导航"部分添加链接。

### 4.2 docs/README.md 目录树

在 `## 文档目录树` 的对应位置新增一行，格式与周围行保持一致：

```
│   ├── 新文件名.md              #   说明
```

### 4.3 docs/README.md 详细目录（可选）

若新文件属于已有的 `#### xxx/` 小节，在该小节的列表中添加一行：

```markdown
- [说明](guide/xxx/新文件名.md) — 一句话
```

---

## Step 5：验证

完成后逐项检查：

- [ ] 文件 locs ≤ 400（运行 `Get-Content 文件路径 | Measure-Object -Line` 确认）
- [ ] 父目录 `README.md` 已更新
- [ ] `docs/README.md` 目录树已更新（新增/删除文件时）
- [ ] 所有内部链接使用相对路径，无断链
- [ ] 代码块已标注语言标识符
- [ ] 无 TODO / 占位文本

---

## 常见场景速查

### 场景：代码重构后同步 API 文档

1. 找到 `reference/` 中对应的文档
2. 更新签名、参数表格
3. 检查 `guide/` 中引用该 API 的示例是否仍然正确

### 场景：新增公开 API

1. 在 `reference/` 的对应文件中添加条目（按字母序或逻辑分组）
2. 若已有文件超出 400 locs，先拆分（新建 `Xb_xxx.md`，将后半内容移入）
3. 若该 API 有代表性用法，在 `guide/` 的对应章节添加示例

### 场景：废弃 API

在 `reference/` 中对应条目前加：

```markdown
> **已废弃（v5.x）**：请改用 [`NewApi`](./xxx.md#new-api)。
```

### 场景：文档文件超出 400 locs

1. 按逻辑将后半内容剪切到新文件（`Xa.主题.md` → `Xa.主题.md` + `Xb.主题.md`）
2. 在原文件末尾添加"延伸阅读"链接指向新文件
3. 更新父目录 `README.md` 和 `docs/README.md`
